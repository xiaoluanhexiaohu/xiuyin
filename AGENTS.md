# AGENTS.md

## Project Goal
Build an offline MVP for one-click singing correction. The system compares a reference vocal and a user vocal, estimates pitch/timing differences, generates a conservative correction plan, exports `corrected_vocal.wav`, `mix.wav`, and `report.json`.

## Directory Structure
- `app/`: FastAPI API layer. It must only call core/job functions.
- `core/`: audio I/O, pitch tracking, features, DTW alignment, correction planning, placeholder rendering, mixing, reporting.
- `jobs/`: local offline batch orchestration and CLI.
- `services/`: optional external services such as source separation adapters.
- `tests/`: pytest unit and integration tests.
- `examples/`: sample manifests.
- `outputs/`: generated artifacts; keep out of source except `.gitkeep`.

## Run Commands
- Install: `pip install -r requirements.txt`
- CLI: `python -m jobs.batch_export --manifest examples/demo_manifest.json`
- API: `uvicorn app.main:app --reload`

## Test Commands
- Run tests: `pytest`
- Quick tests: `pytest -q`

## Code Style
- Python 3.11+.
- Keep modules small, typed, and documented with docstrings.
- Do not hard-code local absolute paths.
- Do not put web framework dependencies inside core algorithms.
- Do not put try/catch blocks around imports; use explicit availability checks such as `importlib.util.find_spec` for optional backends.

## Audio Project Hard Constraints
- Do not use Melodyne as a backend; it is only product inspiration.
- MVP is offline batch correction only, not real-time correction.
- Preserve the user voice by default. Do not imitate celebrities or the original singer's timbre.
- Timbre transfer must be a future explicit opt-in switch.
- Every correction/export result must generate `report.json`.
- Low-confidence regions must be marked and must not be forcibly corrected.
- New modules must include pytest coverage.
- Keep the architecture modular so Demucs, Rubber Band, WORLD, and future JUCE plugins can be added later.

## Third-party License Notes
- `numpy`, `scipy`, `soundfile`, `librosa`, `fastapi`, `pydantic`, and `uvicorn` are generally permissive, but commercial distribution must re-check exact package and transitive dependency licenses.
- `soundfile` depends on `libsndfile`; verify binary distribution obligations.
- Optional `torchcrepe`, Demucs, Spleeter, Rubber Band, and model weights must be reviewed before commercial use.
- Rubber Band may involve GPL/commercial licensing considerations depending on usage/distribution.

## Do Not Do
- Do not implement real-time audio, VST/AU plugins, or neural end-to-end correction in this MVP.
- Do not use `librosa.phase_vocoder` as a production renderer.
- Do not force pitch correction in unvoiced, breath, sibilant, or low-confidence regions.

## Definition of Done
- Dependencies can be installed.
- `pytest` runs.
- CLI reads `examples/demo_manifest.json` and gives clear errors for missing demo audio.
- README explains usage.
- Core modules have docstrings.
- `correction_planner.py` has real testable logic.
- `report.json` is generated by successful jobs.
